Reimplementing Hierarchical Agent Reflection (HAR) in LangGraph

A step-by-step implementation plan for reproducing Wang et al. (2026), “Hierarchical agent reflection for aligning LLM reasoning with clinical diagnostic processes” — with one benchmark (RJUA-QA) selected as the test to complete.

Source paper: Health Information Science and Systems (2026) 14:21 · DOI 10.1007/s13755-025-00410-1 · Plan authored 2026-07-11

TL;DR We rebuild the paper's four-agent clinical-note generation pipeline (ICA → PDA → DDA → Coordinator, with self-reflection loops and hierarchical supervisory correction) as a LangGraph state machine. We do not reproduce the LoRA fine-tuning (needs 8×A100). The one test we complete is diagnostic F1 on RJUA-QA, running HAR at inference to emit a final diagnosis. A fast end-to-end integration test on the paper's own worked case gates correctness before the benchmark run.

1. Scope & assumptions

The paper has three pieces: (A) a hierarchical multi-agent framework that generates structured clinical notes, (B) LoRA SFT of a 7–8B model on those notes, and (C) evaluation on five benchmarks. The framework (A) is the paper's contribution and the natural LangGraph target; (B) requires 8×A100 and is out of scope for a reproducible plan; (C) gives us the test.

Explicit assumptions (please correct if wrong)
  • No fine-tuning. We run the HAR graph at inference time to produce a diagnosis for each RJUA-QA question, instead of distilling notes into a fine-tuned model. This is faithful to the framework and directly testable; the SFT step is documented as optional Phase 2. (In the paper's own ablation this is closest to “Com. HAR” used directly.)
  • LLM = DeepSeek-R1 / Qwen (not Claude). Faithful to the paper, we use DeepSeek-R1 / Qwen as the agent LLM, reached through the OpenAI-compatible API (OPENAI_API_KEY + OPENAI_BASE_URL) via langchain-openai's ChatOpenAI. We deliberately do not use langchain-anthropic. Model id is a config value.
  • Knowledge base subset. The paper's KB covers 433 diseases. We build the 17 urological diseases from Fig. 3 (the differential-diagnosis graph the paper actually draws), which is exactly the domain of RJUA-QA. This keeps the test honest and the KB hand-verifiable.
  • Reflection depth. We use N = 5 reflection iterations for PDA and DDA (the paper's chosen optimum, Fig. 7) and a Coordinator cap of M = 3 correction rounds (paper does not state M; small cap prevents loops).
  • Metric parity. We will not match the paper's fine-tuned F1 (e.g. Qwen2.5-7B-SFT ≈ 46.5 on RJUA-QA); we report our HAR-inference F1 as the completed test and compare qualitatively.

What "done" looks like

A runnable repo where python -m har.run_graph --question "…" emits a complete 9-field clinical note, an integration test reproduces the paper's stress-incontinence worked case, and python -m har.eval_rjua --n 50 prints precision / recall / F1 on an RJUA-QA subset.

3. LangGraph architecture

One graph, one shared state. Nodes are agent steps; conditional edges implement the two self-reflection loops (PDA, DDA) and the hierarchical Coordinator route-back. The Coordinator sits above ICA/PDA/DDA: when it finds a discrepancy against the standardized note, it sends control back to the offending agent with feedback — this is the "hierarchical" part.

            ┌────────────┐
 question → │ ica_extract │  (y1 medical history, y2 physical exam, y3 auxiliary exam)
            └──────┬──────┘
                   ▼
            ┌──────────────┐
            │ ica_summarize│  (y4 clinical features)
            └──────┬───────┘
                   ▼
            ┌──────────────┐   reflect: flag? / error
            │ pda_diagnose │◀────────────────┐   (y5 initial dx, y6 basis)
            └──────┬───────┘                 │
                   ▼                          │ flag==false & iter<N
            ┌──────────────┐                 │
            │ pda_reflect  │──────────────────┘
            └──────┬───────┘  flag==true | iter==N
                   ▼
            ┌──────────────┐
            │ dda_retrieve │  (y7 disease list from KG)
            └──────┬───────┘
                   ▼
            ┌───────────────────┐   reflect: flag? / error
            │ dda_differentiate │◀────────────┐  (y8 differential process)
            └──────┬────────────┘             │
                   ▼                           │ flag==false & iter<N
            ┌──────────────┐                  │
            │ dda_reflect  │──────────────────┘
            └──────┬───────┘  flag==true | iter==N → (y9 final dx)
                   ▼
            ┌──────────────┐
            │  ca_match    │  (fetch standardized note for y9)
            └──────┬───────┘
                   ▼
            ┌──────────────┐   T_flag==false → route back to ica_summarize | pda_diagnose | dda_differentiate
            │  ca_reflect  │─────────────────────────────────┐
            └──────┬───────┘   T_flag==true | ca_iter==M                   │ (feedback carried in state)
                   ▼                                                        ▼
            ┌──────────────┐                                     (offending agent re-runs)
            │   assemble   │ → verified 9-field clinical note + final_diagnosis
            └──────────────┘
Design note — why LangGraph The paper's Algorithm 1 is literally a loop-with-flags state machine. LangGraph's StateGraph + add_conditional_edges expresses the reflect-and-retry loops and the Coordinator's dynamic route-back directly, with the shared state carrying the 9 note fields, iteration counters, and coordinator feedback. Reflection outputs are JSON dicts (exactly as the paper's prompts specify, e.g. {"flag": false, "diagnosis_error": "…"}), parsed from the reply text with a Pydantic model — DeepSeek-R1 is a reasoner without reliable tool-calling, so we do not rely on with_structured_output.

5. Tech stack & global constraints

  • Python 3.11+, langgraph >= 0.2, langchain-openai, openai, pydantic v2, pytest. No langchain-anthropic.
  • Model: deepseek-reasoner (DeepSeek-R1) or a Qwen instruct model for all agents, reached through the OpenAI-compatible API — langchain-openai's ChatOpenAI with base_url=OPENAI_BASE_URL. Set via har/config.py.
  • Auth: OPENAI_API_KEY and OPENAI_BASE_URL in environment, pointed at the DeepSeek or Qwen endpoint. Never hardcode.
  • Reflection I/O contract: every reflect node returns strict JSON matching the paper's prompts (flag + error/final fields). DeepSeek-R1 is a reasoner without reliable tool-calling, so parse the JSON object out of the reply with a Pydantic model (parse_json) — never with_structured_output, never regex.
  • Determinism of graph, not model: iteration caps N=5 (PDA/DDA), M=3 (CA) are hard ceilings so the graph always terminates.
  • Language: RJUA-QA is Chinese; prompts work bilingually. Keep the paper's English prompt text; the patient question carries the language. (Confirm at implementation whether Chinese prompt variants improve extraction.)

import os
from langchain_openai import ChatOpenAI
from pydantic import BaseModel
from har import config

def get_llm(max_tokens: int = config.MAX_TOKENS) -> ChatOpenAI:
    # DeepSeek-R1 / Qwen via any OpenAI-compatible endpoint
    return ChatOpenAI(model=config.MODEL_ID,
                      base_url=os.environ["OPENAI_BASE_URL"],
                      api_key=os.environ["OPENAI_API_KEY"],
                      max_tokens=max_tokens, temperature=0)

def parse_json(schema: type[BaseModel], text: str) -> BaseModel:
    # The reflect prompts already instruct a JSON dict output; extract and validate it.
    s, e = text.find("{"), text.rfind("}")
    return schema.model_validate_json(text[s:e + 1])

from pydantic import BaseModel
from har.llm import get_llm, parse_json
from har.knowledge import key_points
from har import prompts, config

class PDAReflection(BaseModel):
    flag: bool
    diagnosis_error: str | None = None

def pda_diagnose(state):
    msg = prompts.MAKE_PRELIM_DX.format(
        clinical_features=state["clinical_features"], question=state["question"])
    if state.get("pda_error"):
        msg += f"\n\nPrevious reflection found issues, correct them: {state['pda_error']}"
    text = get_llm().invoke(msg).content
    dx, basis = _split_dx_basis(text)
    return {"initial_diagnosis": dx, "diagnostic_basis": basis,
            "pda_iters": state.get("pda_iters", 0) + 1}

def pda_reflect(state):
    kp = key_points(state["initial_diagnosis"])   # retrieve from KB
    msg = prompts.REFLECT_PDA.format(
        preliminary_diagnosis=state["initial_diagnosis"],
        diagnostic_basis=state["diagnostic_basis"], key_inquiry_points=kp)
    r = parse_json(PDAReflection, get_llm().invoke(msg).content)
    return {"pda_error": None if r.flag else (r.diagnosis_error or "unspecified")}

def route_after_pda_reflect(state):
    if state.get("pda_error") and state.get("pda_iters", 0) < config.PDA_MAX_ITERS:
        return "pda_diagnose"
    return "dda_retrieve"

Step 2. har/nodes/dda.py — three nodes:
class DiffProcess(BaseModel):
    diff_process: str

class DDAReflection(BaseModel):
    flag: bool
    final_diagnosis: str | None = None
    diff_error: str | None = None

def dda_retrieve(state):
    dl = differentials(state["initial_diagnosis"])   # from KG
    return {"disease_list": dl}

def dda_differentiate(state):
    kp = {d: key_points(d) for d in state["disease_list"]}
    msg = prompts.DIFFERENTIAL_DX.format(
        diseases_list=state["disease_list"], key_inquiry_points=kp)
    if state.get("dda_error"):
        msg += f"\n\nRe-diagnose these: {state['dda_error']}"
    proc = parse_json(DiffProcess, get_llm().invoke(msg).content).diff_process
    return {"differential_process": proc, "dda_iters": state.get("dda_iters", 0) + 1}

def dda_reflect(state):
    msg = prompts.REFLECT_DIFFERENTIAL.format(
        diseases_list=state["disease_list"],
        differential_diagnosis_process=state["differential_process"])
    r = parse_json(DDAReflection, get_llm().invoke(msg).content)
    if r.flag:
        return {"final_diagnosis": r.final_diagnosis, "dda_error": None}
    return {"dda_error": r.diff_error or "unspecified"}

def route_after_dda_reflect(state):
    if state.get("dda_error") and state.get("dda_iters", 0) < config.DDA_MAX_ITERS:
        return "dda_differentiate"
    return "ca_match"

class CAReflection(BaseModel):
    flag: bool
    ica_error: str | None = None
    pda_error: str | None = None
    dda_error: str | None = None

def ca_match(state):
    return {"_std_note": standardized_note(state["final_diagnosis"])}

def ca_reflect(state):
    raw = _assemble_raw_note(state)
    std = state["_std_note"]
    r = _run_ca_checks(raw, std)   # runs REFLECT_CORRECT_ICA / _PDA / _DDA
    updates = {"ca_iters": state.get("ca_iters", 0) + 1}
    if r.flag:
        updates["ca_target"] = None
    elif r.ica_error:
        updates.update(ca_target="ica", ca_feedback=r.ica_error)
    elif r.pda_error:
        updates.update(ca_target="pda", pda_error=r.pda_error, pda_iters=0)
    elif r.dda_error:
        updates.update(ca_target="dda", dda_error=r.dda_error, dda_iters=0)
    return updates

def route_after_ca(state):
    if state.get("ca_target") is None or state.get("ca_iters", 0) >= config.CA_MAX_ITERS:
        return "assemble"
    return {"ica": "ica_summarize", "pda": "pda_diagnose",
            "dda": "dda_differentiate"}[state["ca_target"]]

def assemble(state):
    return {"clinical_note": _format_note(state)}   # verified 9-field note

Step 1. har/graph.py:
from langgraph.graph import StateGraph, START, END
from har.state import ClinicalNoteState
from har.nodes import ica, pda, dda, coordinator as ca

def build_har_graph():
    g = StateGraph(ClinicalNoteState)
    for name, fn in [("ica_extract", ica.ica_extract), ("ica_summarize", ica.ica_summarize),
                     ("pda_diagnose", pda.pda_diagnose), ("pda_reflect", pda.pda_reflect),
                     ("dda_retrieve", dda.dda_retrieve), ("dda_differentiate", dda.dda_differentiate),
                     ("dda_reflect", dda.dda_reflect), ("ca_match", ca.ca_match),
                     ("ca_reflect", ca.ca_reflect), ("assemble", ca.assemble)]:
        g.add_node(name, fn)
    g.add_edge(START, "ica_extract")
    g.add_edge("ica_extract", "ica_summarize")
    g.add_edge("ica_summarize", "pda_diagnose")
    g.add_edge("pda_diagnose", "pda_reflect")
    g.add_conditional_edges("pda_reflect", pda.route_after_pda_reflect,
                            {"pda_diagnose": "pda_diagnose", "dda_retrieve": "dda_retrieve"})
    g.add_edge("dda_retrieve", "dda_differentiate")
    g.add_edge("dda_differentiate", "dda_reflect")
    g.add_conditional_edges("dda_reflect", dda.route_after_dda_reflect,
                            {"dda_differentiate": "dda_differentiate", "ca_match": "ca_match"})
    g.add_edge("ca_match", "ca_reflect")
    g.add_conditional_edges("ca_reflect", ca.route_after_ca,
                            {"ica_summarize": "ica_summarize", "pda_diagnose": "pda_diagnose",
                             "dda_differentiate": "dda_differentiate", "assemble": "assemble"})
    g.add_edge("assemble", END)
    return g.compile()

Task 9 — RJUA-QA F1 evaluation THE TEST
Step 1. har/eval_rjua.py: for each question, run the compiled HAR graph, take final_diagnosis, normalize to a disease-label set (LLM or KB name-matcher), compare to gold_diseases.
Step 2. Compute set-based precision / recall / F1 per item, then micro- and macro-average across the subset — the paper reports F1 for RJUA-QA (Table 1).
def f1_set(pred: set[str], gold: set[str]) -> tuple[float, float, float]:
    tp = len(pred & gold)
    p = tp / len(pred) if pred else 0.0
    r = tp / len(gold) if gold else 0.0
    f = 2 * p * r / (p + r) if (p + r) else 0.0
    return p, r, f
Step 3. CLI: python -m har.eval_rjua --n 50 prints per-item and aggregate P/R/F1, plus a JSON log of predictions for error analysis. Add --baseline for a single-call comparison (see §7).

7. The one test: RJUA-QA F1

Definition

Run the HAR LangGraph pipeline at inference on an RJUA-QA test subset; for each patient question, take the Coordinator-verified final_diagnosis, map it to disease labels, and score against gold with set-based F1 (micro + macro).

Why this benchmark

  • Domain match: RJUA-QA is urological — exactly the domain of the paper's differential-diagnosis knowledge graph (Fig. 3), so our 17-disease KB is directly applicable.
  • Metric in the paper: Table 1 reports F1 for RJUA-QA, giving a reference point.
  • Open access & open-ended: tests real diagnostic reasoning (not multiple choice), which is what HAR is built for.

Reference points (paper, Table 1)

Qwen2.5-7B-Instruct base F1 = 36.39; the same model SFT'd on HAR notes = 46.50; Baichuan-M1-14B = 50.01. Our HAR-inference number is not directly comparable to a fine-tuned model, but should clearly beat a zero-shot single-call baseline on the same subset — that comparison is the meaningful result.

Built-in baseline

Add --baseline to eval_rjua: one direct LLM call ("diagnose this patient") scored the same way. Reporting HAR-F1 vs baseline-F1 on the identical subset is the honest demonstration that the agentic pipeline helps.

8. Success criteria

  • ✓ Framework fidelity: all four agents, both reflection loops, and the hierarchical Coordinator route-back are present and provably terminating (unit tests per node).
  • ✓ Correctness gate: tests/test_end_to_end.py reproduces the paper's worked case (final diagnosis = stress incontinence + overactive bladder) with all nine note fields populated.
  • ✓ Test completed: python -m har.eval_rjua --n 50 prints P/R/F1 on RJUA-QA and HAR beats the single-call baseline on the same subset.
  • ✓ Reproducible: fresh clone + ANTHROPIC_API_KEY + pip install -e . runs every test.
